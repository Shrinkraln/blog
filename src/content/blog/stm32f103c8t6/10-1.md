# 1. I2C通信协议

1. 连接单片机读写外部模块
2. 相较于串口
   1. rx和tx同时只有一根工作，浪费
   2. 没有应答模式
   3. 只能但对单
   4. 异步时序，单片机中断外部模块不知道
3. 硬件电路
   1. SCL只能主机控制，从机读取；主机推挽，其余上拉或者浮空
   2. SDA不能从机主动控制，必须是主机控制从机读数据，从机才能控制输出
   3. 开漏输出：控制输出高电平，上拉电源打开，但是没有上拉，而且下拉又是断开的，引脚浮空；为了避免输出高电平浮空，外置一个弱上拉电阻（相当于还是推挽输出，但是，传统的推挽输出在SDA上如果没有处理好同时时序，在一端输出高电平，一端输出低电平时会造成短路）
   4. 电路是可以输入，但是不一定接受
   5. 弱上拉高电平：
      1. 配置高电平的始终是外部的上拉电压，当一端上拉一端下拉的时候电流不经过上拉的输出引脚（但是可以经过输入），并且外部上拉电路有电阻保护
      2. 配置低电平，输入引脚被短路，所以**配置高电平时输入引脚的缓冲可以实现共用**，
      3. 线与：只要有一个输出低电平，总线就处于低电平
      4. **写0输出，写1输入**

4. 时序单元
   1. 平静由外部上拉电阻上拉到高电平（**开漏输出1平静状态，因为输出同时只有一个，不妨默认都是输入**）
   2. **SDA早到晚退SCL迟到早退**
   3. 只有主机控制起始
   4. SDA先放数据，SCL再置1，期间读取，依旧是SCL先0，SDA再释放数据
   5. SDA高位先行
   6. SDA放上的电平本身是低电平那CLK怎么知道SDA已经放了数据：不是SDA触发而是CLK协议规定，在CLK上升沿之前SDA必须已经放好了数据，**是SDA配合规定CLK**
   7. 根据协议：**根据CLK的高低电平，结合SDA的电平变化，判断起始停止**（自认为**边沿触发读取**更合理的解释，集合CLK时钟控制存储位的变化）（但是在软件实现I2C通讯，起始条件为了满足复起始条件，SDA都是先拉高，开始时也是先拉低）（**<u>但是始终正确的是SCL低电平写入，高电平读取，当高电平写入0即SDA下拉时就是起始条件，高电平写入1即SDA上拉就是停止条件；这样区分数据传输和终止</u>**；重点是高电平的SDA电平状态判断）
      1. 高电平 下降-开始
      2. 高电平 上升-停止
      3. 低电平 下降-0
      4. 低电平 上升-1
      5. 低电平 不变-不变
   8. 主机发送，接受完一个字节之后，在下一个时钟，接受方会发送一个应答0,（如果主机先接受一个字节，置1，再发送应答，置0）
      1. 主机发送数据时：发送完一个字节，置1，松手，看外部输入：是1，没有设备拉下，没有设备确认收到；是0，有设备拉下
      2. **主机发送数据0-1，接受1-0；两者刚好电平相反**
      3. 在完整的应答中，SDA的电平先是发送端拉低，再由接受端拉低，就是一个连续的低电平；在应答完成之后（在CLK上升沿的时候就已经完成了数据的读取），接收放释放SDA，置1，同时CLK下拉准备接受数据

5. 时序
   1. 每个外设有一个厂家分配的设备地址
   2. 相同设备的最后几位是可以用户更改的
   3. 一般是7位地址+1读写位（10位选择起始位之后两个字节作为地址，并且包含11110+3+7+1，其中11110作为标志位，并且不会在七位中使用
   4. 指定地址写：指定设备（读写使能读，0xD0），指定地址，指定数据（在读模式下主机先控制，再把控制权给从机
   5. 当前地址读：指定设备（读写使能写，0xD1），指定数据（寄存器有指针指向当前地址
   6. 指定地址读：上面两个复合，有重复开始
   7. **能控制CLK的肯定是主设备**，而**从设备是检测接受到的数据是否满足协议**（时钟也因而不是一个客观一直存在的滴答信号，而是**协议标志位**），以及放上数据是否满足CLK0条件，在CKL1后才撤回
   8. **CLK在非终止信号都是0结束**

6. 多返回值函数
   1. 多指针参数
   2. 缓冲区数组（全局变量）
   3. 结构体打包
7. 分层思想
   1. 对寄存器控制：驱动层
   2. 配置引脚，时序：协议层
   3. 获得的数据利用：应用层

```c
void myi2c_start(void){
	myi2c_w_SDA(1);
	myi2c_w_SCL(1);	//为保证SDA拉低时CLK高电平，被检测为起始，先将CLK和SDA拉高，但是CLK高电平时又不允许SDA电平变化，所以先将SDA拉高
	//（CLK高电平来自于每个时序除了停止都是将CLK拉低）
	//SDA高低电平不确定，所以拉高统一
	
	myi2c_w_SDA(0);
	
	myi2c_w_SCL(0);	//起始的判断不是CLK但是要激活CLK输出
}

void myi2c_stop(void){
	myi2c_w_SDA(0);	
	myi2c_w_SCL(1);	//为保证CLK高电平时SDA上升被检测为停止，先将SDA拉低并且将CLK拉高
	
	myi2c_w_SDA(1);
}

void myi2c_sendbyte(uint8_t byte){
	for(uint8_t i=0;i<8;i++){
		myi2c_w_SDA((byte & 0x80>>i)>>7);//读取最高位，非0为1也可以最后不移位
		//CLK低电平放入数据，但是设置的时序都是CLK0结束，不需要再拉低，直接放上数据
		myi2c_w_SCL(1);
		myi2c_w_SCL(0);
	}	
}

uint8_t myi2c_receivebyte(void){
	uint8_t byte=0x00;
	myi2c_w_SDA(1);//等效输入模式
	for(uint8_t i=0;i<8;i++){
		myi2c_w_SCL(1);
		byte=byte|myi2c_r_SDA()<<(7-i);
		myi2c_w_SCL(0);
	}
	return byte;
}

void myi2c_sendack(uint8_t byte){	//响应发送可以主动发送1表示不想在接受
	myi2c_w_SDA(byte);
	myi2c_w_SCL(1);
	myi2c_w_SCL(0);

}

uint8_t myi2c_receiveack(void){
	uint8_t byte=0x00;
	myi2c_w_SDA(1);//等效输入模式
	myi2c_w_SCL(1);
	byte=myi2c_r_SDA();
	myi2c_w_SCL(0);
	
	return byte;
}
```
