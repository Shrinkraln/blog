# 1. DMA数据转换

```c
uint8_t a 0x00;

//主函数中
oled_show(1,1,(uint32_t)&ADC->DR,8);
```

1. 因为是32位机，存储地址最大32位，但是没有用完
2. stm32是使用结构体映射存储器地址
   1. 比如内存4001 2400放的是ADC1寄存器，存放顺序是SR,CR1,CR2，定义一个adctypedef的结构体，那么结构体成员存储顺序是一样的，如果指定结构体的起始地址就是ADC1外设寄存器的起始地址，那么两者将完全重合，访问结构体某个成员就是访问外设寄存器
   2. ADC直接是一个宏定义，对应就是一个地址，利用结构体的偏移量来映射

# 2. ADC和DMA结合

1. ADC本身有两个分组，在规则组中，ADC通道可以指定序列，但是测量结果的存储DR只能储存一个16位结果（也就说明了为什么DMA不需要连续读取），同时ADC规定了连续与否，扫描与否（在多个数据时当然是扫描模式），这个只影响DR的输出结果是循环变化与否；DMA可以指定源端和目标各自的起始地址（变相指定一个组），同时规定是否自动重装，是否连续读取；在打开ADC的DMA通道后，ADC的结果能通过指定通道触发请求，DMA在接收到触发源之后开始搬运数据（DMA开始搬运的三个条件：dma_cmd供电；**触发源**；传输计数器值不为0。而ADC是硬件触发，不是软件随时自由触发的触发源），并且设置ADC到DMA只是作为触发源，还要独立设置DMA通道的源地址
2. 对应关系
   1. ADC扫描模式---DMA连续读取地址
   2. 连续模式（连续读取整个组）---自动重装
