# 1. 串口数据包

1. hex数据包
  1. 包头包尾误判
     	1. 限制载荷数据范围：避免和包头包尾重复
          	2. 使用固定长度数据包
          	3. 增加包头包尾数量
  2. 包头包尾不是全部都需要的
  3. 固定包长和可变选择
     	1. 载荷重复：固定
  4. 数据可以直接转化为字节流
2. 文本数据包：
  1. 代码更加直观
  2. 并且不容易包头包尾误判
  3. 但是解析效率低
3. 接受数据包
  1. 固定包长hex
  2. 不定包长文本（因为是双包尾，有三个状态，单包尾只需要两个状态检测到包尾就可以结束状态了）
  3. 发送的要么没有""的hex码，或是带""的字符串，接受的在缓冲区类型定义为char即可

//状态机的使用
void USART1_IRQHandler(void){
	static uint8_t Rxstate=0;
	static uint8_t pRxstate=0;
	if(USART_GetFlagStatus(USART1,USART_FLAG_RXNE)==SET){
		RX_Data=USART_ReceiveData(USART1);
		if(Rxstate==0){
			if(RX_Data==0xff){
				Rxstate=1;
			}
		}else if (Rxstate==1){
			Serial_RxPacket[pRxstate]=RX_Data;
			pRxstate++;
			//不同于for循环，每次进入状态机只能操作一个数
			if(pRxstate==4){
				Rxstate=2;
				pRxstate=0;
			}
		}else if (Rxstate==2){
			if(RX_Data==0xfe){
				Rxstate=0;
				RX_Flag=1;
            }
		}
		
		
		USART_ClearFlag(USART1,USART_FLAG_RXNE);
	}
}
