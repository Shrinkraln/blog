1. LSI在IWDG软件或者硬件启动后就会自动开启，不用手动开启
2. LSI本身会有误差
3. ENABLE之前可以先喂狗一次，下一次的间隔就是标准间隔
4. 标志位检查不放在循环体，在复位后会重新执行

```C
void IWDG_WriteAccessCmd(uint16_t IWDG_WriteAccess);//写使能
void IWDG_SetPrescaler(uint8_t IWDG_Prescaler);
void IWDG_SetReload(uint16_t Reload);
void IWDG_ReloadCounter(void);//喂狗，自增计数器
void IWDG_Enable(void);
FlagStatus IWDG_GetFlagStatus(uint16_t IWDG_FLAG);

RCC_FlagGet()//可以 查看是WDG复位还是手动复位，而且要手动清除

void WWDG_DeInit(void);
void WWDG_SetPrescaler(uint32_t WWDG_Prescaler);
void WWDG_SetWindowValue(uint8_t WindowValue);//窗口的间隔时间，实际上是喂狗的最短时间间隔，不是W50而是T50-W50
void WWDG_EnableIT(void);
void WWDG_SetCounter(uint8_t Counter);//喂狗，这个八位的counter要包含最高两位都是11，
void WWDG_Enable(uint8_t Counter);//自减计数器需要再确认值，因为没有设置重装载值，需要在使能的时候也传入COUNTER并且，这个八位的最高位是1，使能，启动WDG之后不会被关闭，所以SETCOUNTER的时候最高位可以写0
FlagStatus WWDG_GetFlagStatus(void);
void WWDG_ClearFlag(void);
```
